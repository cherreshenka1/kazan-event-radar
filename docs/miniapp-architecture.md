# Kazan Event Radar: структура продукта

## Главная идея

Лучший формат - гибрид:

- Telegram Bot: команды, уведомления, напоминания, согласование постов менеджером, публикации в канал.
- Telegram Mini App: красивая витрина, поиск, категории, избранное, маршруты, билеты, личный план.
- Channel Publisher: готовит посты для канала по расписанию и публикует только после approval.
- Source Scanner: собирает события и места из Telegram, RSS, HTML, соцсетей и экспортов/API.

## Mini App разделы

- Афиша: события сегодня/на неделю, кнопка билетов/источника, избранное, напоминания.
- Места: парки, достопримечательности, где покушать, отели, экскурсии.
- Маршруты: легкие, средние, сложные пешие маршруты.
- Избранное: личный план пользователя из мест, маршрутов и мероприятий.

## Напоминания

Для событий с распознанной датой создаются два напоминания:

- за сутки до начала;
- за час до начала.

Если дата не распознана из источника, Mini App не создает напоминание автоматически. Следующий этап - добавить ручной выбор даты пользователем.

## Билеты

В MVP кнопка “Билеты/подробнее” ведет на исходную ссылку события. Следующий слой - `ticketLinks[]`, где можно хранить несколько площадок:

- официальный сайт события;
- агрегатор;
- касса;
- партнерская ссылка;
- промо/реферальная ссылка.

## Канал

Расписание черновиков:

- бот готовит 10 черновиков каждый день в 09:00 по Москве;
- менеджер проверяет каждый черновик;
- посты попадают в канал только после ручной кнопки “Опубликовать”;
- менеджер сам выбирает, когда именно опубликовать каждый пост.

Cron: `0 9 * * *`.

Количество черновиков задается переменной `DRAFTS_PER_DAY=10`.

Каждый черновик сначала уходит менеджеру в `TELEGRAM_MANAGER_CHAT_ID`. В канал пост попадает только после кнопки “Опубликовать”.

## Что нужно для продакшена

- HTTPS URL для Mini App.
- Настройка Mini App URL в BotFather.
- Бот должен быть админом канала.
- Менеджер должен открыть бота и написать `/start`, чтобы бот мог отправлять approval-сообщения.
- Если известен только username менеджера, задайте `TELEGRAM_MANAGER_USERNAME`. После `/start` или `/id` бот сам сохранит числовой chat id менеджера в `data/runtime.json`.
- Если канал приватный и есть только invite link, задайте `TELEGRAM_CHANNEL_INVITE_URL`, добавьте бота администратором канала и отправьте в канал `/channelid`. Бот сохранит числовой channel id в `data/runtime.json`.
- Желательно заменить JSON-хранилище на SQLite/PostgreSQL, когда пользователей станет больше.
