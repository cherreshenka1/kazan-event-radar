# Какие данные нужны для источников

Не присылайте пароли от личных аккаунтов в чат. Для продакшена лучше использовать официальные API, отдельные сервисные аккаунты и токены с минимальными правами.

## Instagram

Лучший путь: Instagram API через Meta.

Что понадобится:
- Instagram Business или Creator account.
- Facebook Page, связанная с этим Instagram-аккаунтом.
- Facebook/Meta Developer account.
- Meta App с настроенным Facebook Login или Instagram Login.
- Доступ пользователя, который может управлять связанной Facebook Page.
- Permissions для чтения медиа, минимум по официальному Facebook Login flow: `instagram_basic`, `pages_show_list`.
- Page ID и Instagram Business/Creator Account ID.
- Access token, лучше long-lived, с понятным сроком обновления.
- Список аккаунтов/профилей, которые разрешено мониторить.

Что не нужно:
- пароль от Instagram;
- личный аккаунт без разрешения владельца;
- обход антибот-защиты и неофициальный scraping закрытых данных.

## TikTok

Лучший путь: TikTok for Developers + Display API.

Что понадобится:
- TikTok Developer account.
- TikTok app, прошедшее approval для Login Kit и нужных API products.
- Scopes: `user.info.basic`, `video.list`.
- OAuth access token от аккаунта, чьи видео нужно читать.
- Список аккаунтов/источников, которые разрешено мониторить.

Важно:
- `cover_image_url` у TikTok может быть временным, поэтому его нужно периодически обновлять через API.
- Для чужих аккаунтов без авторизации нужны отдельные разрешенные механизмы, а не логин/пароль.

## Telegram

Что понадобится:
- публичные каналы: username канала, например `@kuda_go_kazan`.
- закрытые каналы: нужно добавить бота/сервисный аккаунт с разрешениями, если это законно и согласовано с владельцем.
- для публикации в ваш канал: `TELEGRAM_CHANNEL_ID`, бот должен быть администратором канала с правом публикации.
- для согласования менеджером: `TELEGRAM_MANAGER_CHAT_ID`.
- если у вас есть только username менеджера: `TELEGRAM_MANAGER_USERNAME`, после сообщения менеджера боту chat id сохранится автоматически.
- если канал приватный и есть только invite link: `TELEGRAM_CHANNEL_INVITE_URL`, после добавления бота админом отправьте в канал `/channelid`.

## Сайты и статьи

Для каждого сайта лучше собрать:
- URL главной страницы афиши/статей;
- RSS feed, если есть;
- sitemap URL, если есть;
- разрешенные категории: события, рестораны, парки, маршруты, отели;
- CSS-селекторы для HTML-парсинга: карточка, заголовок, ссылка, дата, описание, изображение;
- правила частоты запросов, чтобы не перегружать сайт;
- список запрещенных источников, если их не нужно парсить.

## Билетные площадки

Что понадобится:
- список площадок, которые можно показывать как ticket links;
- партнерские/реферальные параметры, если они есть;
- API-ключи, если площадка предоставляет API;
- правила приоритета ссылок: официальный сайт мероприятия, агрегатор, касса, партнер.

## Минимальный набор от вас для старта

- Новый Telegram bot token уже локально в `.env`.
- `TELEGRAM_CHANNEL_ID` для канала, куда публиковать посты.
- `TELEGRAM_MANAGER_CHAT_ID` менеджера, который согласовывает посты.
- Или минимум: `TELEGRAM_MANAGER_USERNAME` + сообщение менеджера `/start` боту, а для приватного канала - добавить бота админом и отправить `/channelid` в канал.
- Список Telegram-каналов Казани, которые можно мониторить.
- Список сайтов/афиш/медиа, которые хотите включить.
- Если подключаем Instagram: Business/Creator аккаунт + связанная Facebook Page + Meta App/token.
- Если подключаем TikTok: TikTok Developer app + OAuth access token.
- Список билетных сайтов и партнерских ссылок, если они есть.
