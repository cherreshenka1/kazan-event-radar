# Что нужно от тебя для подключения источников

Не присылай пароли в чат. Для проекта лучше использовать отдельные сервисные аккаунты и токены.

## 1. Telegram

Уже почти готово. От тебя нужен только список каналов, которые можно мониторить.

Что прислать:
- username канала, если он публичный;
- invite link и подтверждение, что бот может быть добавлен админом, если канал приватный;
- какие категории у канала: события, рестораны, бары, маршруты, парки, отели, секретные места.

## 2. Instagram

Нужен отдельный проектный аккаунт, не личный.

Что создать:
- Instagram Business или Creator аккаунт проекта;
- Facebook Page, связанная с этим Instagram;
- Meta Developer account;
- Meta App.

Что потом понадобится для подключения:
- long-lived access token;
- Instagram Account ID;
- Facebook Page ID.

Что важно:
- через официальный Instagram API мы можем работать только с теми аккаунтами, к которым есть законный доступ через Meta;
- логин и пароль Instagram мне не нужны.

## 3. TikTok

Тоже лучше создать отдельный проектный аккаунт.

Что создать:
- TikTok аккаунт проекта;
- TikTok Developer account;
- приложение в TikTok for Developers.

Что потом понадобится:
- OAuth access token;
- Open ID пользователя, если он понадобится для конкретного потока;
- подтверждение, что этот аккаунт можно использовать как источник.

Что важно:
- официальный `video.list` API работает с токеном авторизованного пользователя и его публичными видео;
- логин и пароль TikTok мне не нужны.

## 4. Отзывы ресторанов

Тут есть два нормальных пути.

Вариант A. Быстрый и практичный:
- дашь список ресторанов;
- для каждого ресторана добавим ссылки на карточки в Яндекс Картах, 2ГИС, Google Maps, TripAdvisor или другом источнике отзывов.

Вариант B. Сильнее и точнее:
- дашь доступ к одному-двум стабильным источникам отзывов;
- мы будем собирать рейтинг, число отзывов и короткую выжимку по ним.

Лучше всего для старта:
- 2ГИС API key;
- ссылки на карточки ресторанов в Яндекс/2ГИС;
- список ресторанов, которые для тебя приоритетны.

Важно:
- если ресторан нам не принадлежит, то привилегированный доступ к внутренней аналитике отзывов обычно недоступен;
- зато мы можем легально работать с публичными карточками и официальными API, где это разрешено.

## 5. Сайты и медиа

Что нужно от тебя:
- список сайтов, которые точно можно использовать;
- если есть, RSS или разделы афиши;
- если есть особые приоритеты: только Казань, только апрель 2026, только будущие события.

## 6. Билеты

У нас уже добавлены:
- Yandex Афиша
- Афиша
- Kassir.ru Казань
- МТС Live Казань

От тебя может понадобиться:
- партнерская ссылка, если она есть;
- список площадок, которые нужно ставить выше остальных.

## 7. Что тебе лучше сделать прямо сейчас

Если хочешь двигаться быстро, лучший набор на ближайший шаг такой:
- создать отдельный Instagram Business/Creator аккаунт проекта;
- создать отдельный TikTok аккаунт проекта;
- собрать таблицу ресторанов со ссылками на Яндекс/2ГИС;
- прислать список Instagram-аккаунтов и TikTok-аккаунтов, которые мы хотим мониторить;
- прислать список новых Telegram-каналов и сайтов.

## 8. Почему Mini App сейчас открывает старую черную страницу

Если в Telegram видно `no tunnel here :(`, значит бот или BotFather все еще открывает старую временную ссылку, а не наш текущий GitHub Pages URL.

Правильная ссылка сейчас такая:

`https://cherreshenka1.github.io/kazan-event-radar/`

Ее нужно поставить:
- в BotFather -> Mini Apps -> Direct Link;
- в BotFather -> Bot Settings -> Menu Button;
- и после этого заново открыть бота и нажать новую кнопку, а не старую из старого сообщения.
